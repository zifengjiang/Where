# 家庭物品定位 iOS App 设计规格

## 目标

构建一款完全离线的 iOS App。用户可以拍摄或选择家中场景照片，在照片上用定位点记录多个物品，并通过物品名称、别名或标签搜索。选择物品后，App 在“所有物品”页面上方显示对应场景照片并高亮其定位点，同时展示可选的物品外观照片，帮助用户同时确认“东西是什么”和“东西在哪里”。

## 范围

### MVP 包含

- 数据仅保存在当前 iPhone，不提供 iCloud 同步或家庭共享。
- 支持相机拍摄和从系统照片选择器选择场景照片。
- 一张场景照片可以放置多个物品定位点。
- 每个物品包含名称、多个别名、多个标签、位置说明、普通多行备忘和可选外观照片。
- 外观照片支持主体分割，并以可翻面的剪影卡片展示。
- 搜索覆盖名称、别名和标签，支持标准化后的子串匹配。
- 场景、物品、定位点及其资料均可编辑和删除。
- 最低支持 iOS 26。

### MVP 不包含

- iCloud、多设备同步、家庭共享或账号体系。
- 自动图像识别、物品名称推荐或自动定位。
- 拼写纠错、语义搜索或 AI 搜索。
- 一件物品同时存在于多个场景。

## 品牌与 App Icon

- 产品品牌名和设备桌面显示名为 **Where**。
- Xcode 工程使用不依赖商店名称唯一性的内部工程名；Bundle Identifier 在创建工程时使用开发者自己的反向域名配置。
- App Store 已存在相近名称。提交发布前必须先在 App Store Connect 验证名称可用性。如果精确的 `Where` 无法占用，App 内品牌和设备桌面显示名仍保持 `Where`，商店列表名称使用带描述后缀的可用名称，例如 `Where · Home Locator`。最终商店名称需在发布前再次搜索并确认，不把示例后缀视为已保证可用。
- Icon 采用“场景角落/柜体轮廓 + 定位点”概念：暖橙色背景、深墨绿色几何轮廓、暖红色定位点和浅色描边。
- Icon 保持大形状、少元素、无文字，确保在主屏幕小尺寸和设置列表中仍可识别。
- 最终 Icon 使用 1024×1024 画布分别导出背景、场景轮廓、定位点和描边矢量图层，并在 Apple Icon Composer 中生成单个多层 `.icon` 文件。
- 在 Icon Composer、Simulator 和真机上验证 Default、Dark、Clear 与 Tinted/Mono 外观；动态材质、模糊、阴影和最终圆角遮罩交给系统处理，不预绘制外层圆角或静态高光。

## 信息架构

App 使用两个主要 Tab，并提供独立的添加动作：

1. **场景**：以照片卡片展示全部场景。卡片包含场景名称、缩略图和物品数量。进入场景后可以查看、添加、移动、编辑或删除定位点。
2. **所有物品**：上方显示当前选中物品所属的场景照片与高亮定位点；下方显示搜索框和物品列表。列表项包含可选的主体分割缩略图、名称和标签。点击不同物品只更新本页上方照片区域，不导航到独立详情页。选中物品的外观以可翻面的剪影卡片呈现：正面是去背景后的物品，背面是在物品轮廓内排版的备忘与记录时间。
3. **添加**：使用右下角独立按钮发起录入。两个 Tab 在左侧成组，添加按钮在右侧与导航分离并始终处于同一底部平面。最终产品决策使用 iOS 26 的独立 action-tab 视觉（Search role 承载动作并拦截选择），触发后立即恢复原内容 Tab，且为辅助技术保留“添加场景”的动作语义。

“所有物品”页面未选择物品时，上方区域显示引导文案“选择一个物品查看它的位置”；首次加载且列表非空时不自动选择，避免用户误以为首项被搜索命中。

## 核心流程

### 创建场景并标记物品

1. 用户点击右下角“＋”。
2. App 立即打开 Apple 系统相机；模拟器或无相机设备直接打开系统照片选择器。权限拒绝时提供设置和相册恢复入口。
3. App 读取照片、修正方向并压缩，用户填写必填的场景名称。
4. App 打开全屏场景标记界面。
5. 用户点击照片放置定位点。
6. App 打开物品编辑面板。名称必填；别名、标签、位置说明、普通多行备忘和外观照片可选。
7. 外观照片通过单独拍摄或从相册选择添加，不从场景照片裁剪。
8. App 使用 VisionKit 分析外观照片。默认选中面积最大的主体；识别到多个主体时，用户可点击目标切换。确认后保存压缩原图和透明背景主体图。
9. 保存物品后返回同一场景标记界面，用户可继续添加、移动或删除定位点。
10. 用户点击“完成”，App 在单个数据库事务中保存场景和全部物品。`createdAt` 记录首次创建时间，后续修改只更新 `updatedAt`。
11. 保存成功后返回“场景”Tab 并显示新场景；保存失败则保留草稿并允许重试。

### 搜索并定位物品

1. 用户进入“所有物品”Tab。
2. 用户输入物品名称、别名或标签的一部分。
3. 列表实时显示匹配结果。
4. 用户点击某一物品。
5. 页面上方切换到该物品所属的场景照片，只高亮目标物品的定位点，并显示物品名称和位置说明。
6. 列表项和物品卡片正面的主体图帮助用户确认物品长相；上方场景照片帮助用户确认存放位置。
7. 用户点击物品卡片后以轻量 3D 动画翻到背面，查看限制在主体剪影轮廓内的备忘和记录时间；再次点击返回正面。
8. 用户点击背面的编辑动作，在标准矩形 Sheet 中编辑普通多行文本，保存后重新生成剪影内排版。

## 技术架构

### 技术栈

- Swift 6
- SwiftUI
- iOS 26 SDK
- GRDB 7.x，通过 Swift Package Manager 集成
- PhotosUI `PhotosPicker`
- 相机拍摄使用系统相机界面的 SwiftUI 封装
- XCTest 与 XCUITest

### 分层

- **App/UI**：SwiftUI 页面、路由、Sheet 和系统照片/相机入口。
- **Feature state**：每个主要流程使用聚焦的 `@MainActor` Observable ViewModel，负责页面状态和用户意图，不直接执行 SQL 或文件操作。
- **Repository**：为场景、物品和搜索提供面向用例的接口，组合数据库事务与图片存储。
- **Database**：`AppDatabase` 持有一个长期存在的 GRDB `DatabaseQueue`，运行迁移并提供读写入口。
- **Image processing and storage**：`SubjectSegmentationService` 使用 VisionKit 获取候选主体和透明主体图；`ImageStore` 负责临时文件、方向校正、压缩、正式文件移动、替换和删除。
- **Silhouette layout**：`SilhouetteTextLayout` 从主体 Alpha 蒙版生成简化且向内收缩的安全排版路径，并使用 Core Text 计算只位于轮廓内部的只读文本布局。

第一版使用 `DatabaseQueue`。数据量和并发需求很小，不引入 `DatabasePool`。GRDB `ValueObservation` 将场景及物品查询变化传递给 ViewModel，SwiftUI View 不直接观察数据库细节。

## 数据模型

### `scene`

- `id`: Text，UUID 字符串，主键
- `name`: Text，非空
- `imagePath`: Text，非空，Application Support 下的相对路径
- `createdAt`: Date，非空
- `updatedAt`: Date，非空

### `item`

- `id`: Text，UUID 字符串，主键
- `sceneID`: Text，非空，外键关联 `scene.id`，删除场景时级联删除
- `name`: Text，非空
- `locationNote`: Text，可空
- `appearanceOriginalImagePath`: Text，可空，压缩后的物品原图
- `appearanceCutoutImagePath`: Text，可空，带透明背景的主体图
- `note`: Text，可空，普通多行备忘
- `normalizedX`: Double，非空，范围 `0...1`
- `normalizedY`: Double，非空，范围 `0...1`
- `createdAt`: Date，非空
- `updatedAt`: Date，非空

### `itemAlias`

- `id`: Text，UUID 字符串，主键
- `itemID`: Text，非空，外键关联 `item.id`，级联删除
- `value`: Text，非空
- 同一物品内标准化后的别名唯一

### `tag`

- `id`: Text，UUID 字符串，主键
- `name`: Text，非空
- `normalizedName`: Text，非空且唯一

### `itemTag`

- `itemID`: Text，外键关联 `item.id`，级联删除
- `tagID`: Text，外键关联 `tag.id`，级联删除
- `(itemID, tagID)` 为联合主键

数据库启用外键约束，并通过 `DatabaseMigrator` 管理版本。定位点保存为相对于原始照片显示区域的归一化坐标，而不是屏幕像素；点击只在实际照片内容范围内生效，Aspect Fit 产生的留白不接受定位点。

## 物品剪影卡片

- 正面以透明主体图为主要内容，不显示原始照片背景。
- 背面复用主体 Alpha 蒙版形成剪影卡片，使用暖色纸张式填充和足够对比度的文字。
- `SilhouetteTextLayout` 将蒙版转为简化路径，剔除过小的孤立区域，并向内收缩安全边距。Core Text 在该路径内部自动换行。
- 背面优先显示备忘正文，并在空间允许时显示本地化的 `createdAt`，例如“记录于 2026年7月11日 21:30”。
- 正文超过轮廓容量时显示可见内容和省略提示；点击背面可进入完整备忘 Sheet，不在不规则轮廓内实现滚动或直接编辑。
- 如果轮廓过窄、碎片化、排版面积不足或布局计算失败，使用主体外接框内的圆角卡片作为安全回退。回退不影响备忘编辑和完整内容查看。
- 动画遵循“减少动态效果”设置；启用该设置时使用淡入淡出替代 3D 翻转。

## 搜索语义

第一版的“模糊匹配”明确为标准化子串匹配，不包含错别字纠正。查询覆盖：

- `item.name`
- `itemAlias.value`
- `tag.name`

标准化包括去除首尾空白、Unicode 兼容标准化、拉丁字符小写化和全角/半角兼容。空搜索返回全部物品，按最近更新时间降序排列；非空搜索优先显示名称命中，其次别名命中，再其次标签命中，同等级按最近更新时间排序。所有 SQL 使用参数绑定。

## 图片存储

- SQLite 仅保存图片相对路径，不保存大尺寸二进制 Blob。
- 图片存放于 App 私有 `Application Support/Images` 目录。
- 录入过程的图片先写入 `tmp` 草稿目录。
- 保存前修正 EXIF 方向，限制最长边并压缩为适合屏幕查看的格式。
- 物品外观照片同时保留压缩原图和透明主体图，以便用户以后重新选择主体；透明主体图使用支持 Alpha 的格式。
- VisionKit 返回多个主体时，按主体边界面积选择默认项，并允许用户点击候选主体切换。
- 数据库事务成功后将草稿图片移动到正式目录；事务失败或用户取消时清理草稿。
- 替换图片时先完成新文件和数据库记录写入，再删除旧文件。
- 删除记录时先提交数据库事务，再删除已不被引用的文件。
- App 启动后执行一次轻量孤立文件扫描，只删除超过安全时间窗口且没有数据库引用的正式图片和遗留草稿。

## 错误处理

- 相机权限拒绝：显示解释、打开系统设置入口和“改从相册选择”。
- 照片读取或压缩失败：保留当前录入状态，指出失败步骤并允许重新选择。
- 主体分析不受支持、没有识别结果或生成透明图失败：说明未能分离背景并使用原图继续保存，不阻塞物品录入。
- 剪影文字路径不可用：自动采用圆角卡片回退布局，不丢失备忘内容。
- 数据库保存失败：保留完整草稿和临时图片，提供重试与取消。
- 图片文件缺失或损坏：显示占位图；文字记录仍可搜索、编辑和删除。
- 删除场景：确认框明确说明会删除该场景内全部物品；数据库失败时不删除图片。
- 所有长耗时图片处理离开主线程执行，界面显示忙碌状态并阻止重复提交。

## 编辑和删除

- 场景可修改名称或替换照片。替换场景照片时保留已有归一化定位点，但保存前向用户确认新照片可能使标记位置不再准确，并提供逐点调整。
- 物品可修改名称、别名、标签、位置说明、普通多行备忘、外观照片、所选主体和定位点。
- 删除物品会删除别名和标签关联；没有任何物品引用的标签会在事务中清理。
- 删除场景会级联删除所属物品、别名和标签关联，并在事务成功后清理相关图片。

## 无障碍

- 所有按钮的可点击区域至少为 44×44 点。
- 支持 Dynamic Type，重要文字不以固定高度裁切。
- 定位点的无障碍标签包含物品名称，提示包含位置说明。
- 外观照片和场景照片提供有意义的无障碍描述。
- 颜色不是区分选中定位点的唯一手段；选中点同时改变大小、描边和标签。

## 测试策略

### 单元测试

- 在内存数据库中验证全部迁移。
- 验证场景、物品、别名和标签的增删改查与级联删除。
- 验证名称、别名、标签、中文子串、空查询、特殊字符和排序规则。
- 验证归一化坐标在不同图片宽高比、容器尺寸和缩放状态下的双向转换。
- 验证图片草稿、保存、替换、取消、失败回滚和孤立文件清理。
- 验证多主体默认选择、点击切换、透明主体图生成、重新分割和识别失败回退原图。
- 验证剪影路径简化、向内边距、自动换行、文本溢出、省略提示和圆角卡片安全回退。
- 验证 `createdAt` 在编辑后保持不变，`updatedAt` 正确更新，并按区域设置格式化显示。
- 验证选择物品只高亮目标定位点，并正确切换所属场景。

### UI 测试

- 添加一个场景并连续标记多个物品。
- 为单个物品添加外观照片。
- 在包含多个主体的外观照片中切换目标主体，并验证保存后的透明背景效果。
- 通过名称、别名和标签搜索并选择物品。
- 验证同一“所有物品”页面上方显示正确场景和高亮定位点。
- 验证物品卡片正反面切换、剪影内备忘、完整备忘 Sheet 和减少动态效果模式。
- 验证相机拒绝、图片缺失、无搜索结果和删除确认界面。

### 验收标准

用户可以完全离线地记录多个场景和物品；每张场景照片可包含多个定位点；物品可拥有经主体分割的独立外观照片、普通多行备忘和不可变的首次记录时间；用户可以通过名称、别名或标签的子串找到物品，在“所有物品”页面内看到它的外观以及其在场景照片中的高亮位置，并通过翻转剪影卡片查看轮廓内备忘。
